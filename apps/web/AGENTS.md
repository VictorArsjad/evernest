# AGENTS.md — `apps/web` (Vite + React PWA)

Scoped notes for the frontend. Read the **repo root `AGENTS.md`** first for the
big picture (prod topology, data-model contract, mobile safe-area rules). This
file only covers FE-local conventions.

## Stack

Vite 6 + React 18 + TypeScript, TanStack Router (file-based) + TanStack Query,
Tailwind, Zustand for auth state, `idb` for the offline outbox, `vite-plugin-pwa`
(Workbox). Forms use `react-hook-form` + `zod`. Charts use `recharts`.

## Layout

```
src/
  main.tsx            app bootstrap (router + query client + auth bootstrap)
  routeTree.gen.ts    GENERATED by `tsr generate` — never hand-edit
  routes/             file-based routes (see naming below)
  lib/                non-UI logic, each with a .test.ts sibling
  components/         shared presentational components
  styles/             Tailwind entry + global css
public/               PWA manifest, icons (regen via `npm run icons`)
scripts/              build-time helpers (icon generation)
```

## Route file naming (TanStack file-based routing)

- `__root.tsx` — app shell. Already pads for iOS/Android safe areas; see root
  `AGENTS.md` before adding any `fixed`/`sticky` UI.
- `_app.*` — routes behind the authenticated app layout (`_app.tsx`). E.g.
  `_app.log.bottle.tsx` → `/log/bottle`, `_app.charts.tsx` → `/charts`.
- `_auth.*` — public auth layout (`_auth.login.tsx`, `_auth.register.tsx`).
- `invite.$token.tsx` — dynamic param route.
- After adding/renaming a route file, the route tree regenerates automatically
  (`predev` / `prebuild` / `pretypecheck` run `tsr generate`). If editing
  outside those scripts, run `npm run routes`. Don't edit `routeTree.gen.ts`.

## Data access — the seams that matter

All server I/O goes through `lib/api.ts` and is wired into TanStack Query in
`lib/queries.ts`. Two entrypoints, pick deliberately:

- **`api<T>(path, opts)`** — typed fetch. Attaches the in-memory access token,
  does a single 401→refresh→retry, throws `ApiError` on non-2xx. Use for
  **reads (GET)** and for auth calls.
- **`apiQueued<T>(path, { idempotencyKey, synthesize, … })`** — the **mutation
  seam**. Tries the real request first; on network failure / 5xx / expired-auth
  it enqueues into the offline outbox and returns the caller's `synthesize()`
  result so optimistic cache updates still run. Use for **every write**.

**Never bypass the outbox for writes.** New event kinds add a mutation hook in
`queries.ts` that:
1. generates the row id client-side via `makeId()` (`crypto.randomUUID()`),
2. calls `apiQueued` with that id in the body + an `idempotencyKey`,
3. injects the row via `upsertList` in `onSuccess` so the Today list updates
   immediately, then invalidates to refetch when online.

This works because the BE accepts client ids and does `ON CONFLICT (id) DO
NOTHING`, so replay is idempotent (see `lib/outbox.ts` header for the full
contract). `lib/outbox.ts` is import-cycle-free leaf code — it receives its
dispatcher via `setDispatcher` (wired at the bottom of `api.ts`); don't import
`api` into `outbox`.

## Units & timezones

- DB is canonical `ml` / `cm` / `grams` / UTC. `lib/units.ts` is the **only**
  place canonical values get converted to the user's display unit. Keep new
  formatters there, framework-agnostic (no React/hooks/date-fns), with a
  `units.test.ts` case. Conversion constants are named (`ML_PER_FL_OZ`, etc.).
- List endpoints take RFC3339 `from`/`to`; compute the window in the user's
  timezone on the FE (`date-fns` / `date-fns-tz`), send UTC to the API.

## Auth state

`lib/authStore.ts` (Zustand) holds the in-memory access token and persists the
refresh token in `localStorage` (NOT a cookie — iOS Safari ITP blocks the
cross-site github.io↔ts.net cookie). `bootstrapAuth()` runs on mount. On
login success, call `kickAfterReauth()` so outbox records parked on a
401 backoff drain immediately.

## API base URL

`VITE_API_BASE_URL` selects same-origin (dev/Vite proxy, leave unset) vs
absolute (GH Pages → ts.net API). `api.ts` appends `/v1`.

## Checks before done

`make web-build` (runs `tsc -b` + `vite build`) and `npm run lint` + `npm run
test` (Vitest, uses `fake-indexeddb` for outbox tests). These match CI; don't
push without them green. Every `lib/*.ts` module ships with a `.test.ts`
sibling — keep that up.
